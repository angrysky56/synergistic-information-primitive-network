# Contributing to SIP-Net

Thank you for your interest in improving SIP-Net! We value contributions that enhance the mathematical stability, interpretability, or scaling capabilities of the network.

## Development Standards

### 1. Mathematical Rigor
All new estimators must be validated against canonical implementations. If you add a new metric, include a derivation or reference in the docstring.

### 2. Type Safety
We use strict typing. Every function must have type hints. Run `uv run mypy src/sipnet` before submitting.

### 3. Documentation
Update the corresponding files in `docs/` if you change the API or architecture. We use GSD-Doc-Writer standards.

## Submission Process

1. **Fork and Branch**: Create a descriptive branch (e.g., `feat/add-renyi-alpha`).
2. **TDD Workflow**: Add a failing test for your feature before implementing.
3. **Lint and Test**: Ensure all tests pass and `mypy` is clean.
4. **Pull Request**: Provide a detailed description of the information-theoretic impact of your change.

Check [.planning/ROADMAP.md](.planning/ROADMAP.md) and [.planning/STATE.md](.planning/STATE.md) for current priorities.
