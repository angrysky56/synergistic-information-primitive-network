# SIP-Net: Configuration Guide

## Model Configuration

The `SIPNet` architecture is configured via its constructor, typically through a hyperparameter dictionary.

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `hidden_dim` | `int` | `32` | Width of the IPP layers. |
| `num_layers` | `int` | `1` | Depth of the hierarchical stack. |
| `hf_model_name` | `str` | `None` | Hugging Face model identifier for sensory encoding. |
| `hf_freeze` | `bool` | `True` | Whether to disable gradients for the HF encoder. |
| `dropout` | `float` | `0.1` | Dropout rate for hidden states. |

## Training Configuration

The `CognitiveTrainer` uses phase-based configuration to stabilize training.

### Synergy Weights
| Objective | Default Weight | Description |
|-----------|----------------|-------------|
| `synergy` | `0.1` | Encourages interaction-based information. |
| `transfer_entropy` | `0.05` | Directs information flow. |
| `ais` | `0.01` | Stabilizes node memory. |
| `inter_layer_syn` | `0.05` | Discourages redundancy between layers. |

### Cognitive Phasing
- **Phase 1**: Base layer training. Freeze layers > 0.
- **Phase 2**: Middle-tier engagement. Unfreeze layers 1-2.
- **Phase 3**: Full hierarchical optimization. All layers active.
