# SIP-Net: Development Guide

## Environment Setup

The project uses `uv` for environment and dependency management.

```bash
# Create and sync environment
uv venv
source .venv/bin/activate
uv pip sync
```

## Project Structure

- `src/sipnet/domain`: Core information-theoretic primitives and network graph.
- `src/sipnet/application`: Training logic, loss functions, and execution heads.
- `src/sipnet/infrastructure`: Data loaders, metrics, and external integrations.
- `scripts/`: Entry points for training and experimentation.

## Training Workflow

### 1. Synthetic Verification
Always verify new architectural changes on synthetic tasks first.
```bash
uv run python scripts/train_nlp_coreference.py
```

### 2. Large-Scale Training
For OntoNotes 5.0, use the multi-layer training script.
```bash
uv run python scripts/train_ontonotes_coref.py
```

## Adding New Primitives
To add a new information-theoretic metric:
1. Implement the estimator in `src/sipnet/domain/metrics/`.
2. Integrate it into `SIPLayer.forward`.
3. Add corresponding weights to `CompositeLoss`.
